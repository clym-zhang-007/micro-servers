# chunking

文本切片（Chunking）策略示例：同一批文本可用多种方式切块，便于对比效果或接入 RAG / 向量化流水线。

## 目录说明

- `1-固定长度切片.py`：目标长度 + 重叠，在句末/分号附近微调截断
- `2-句子边界切片.py`：先按句拆再合并，尽量不切在句中
- `3-LLM语义切片.py`：调用大模型按语义切块（需 API）
- `4-层次切片.py`：面向 Markdown 等标题层级，小节内再控长
- `5-滑动窗口切片.py`：固定窗口 + 步长，可控制重叠
- `6-自适应切片.py`：按空行段落合并，长度带容差
- `切片策略对比测试.py`：统一加载 1～6 号模块并打印对比

## 环境准备

```bash
python -m pip install --upgrade pip
python -m pip install -r chunking/requirements.txt
```

使用 **3-LLM语义切片** 或对比脚本中的 LLM 分支时，需配置环境变量（与脚本内 DashScope 兼容接口一致）：

```powershell
$env:DASHSCOPE_API_KEY="your_key"
```

## 运行方式

在仓库根目录：

```bash
python chunking/1-固定长度切片.py
python chunking/切片策略对比测试.py
```

单独运行各脚本会执行内置示例文本；对比脚本会依次调用六种策略并输出统计与样例块。

## 策略选型：什么场景适合用哪种切片

以下按**数据形态、目标与约束**选择策略，可组合使用（例如先层次再句界）。

### 1. 固定长度（`1-固定长度切片.py`）

**适合：**

- 长文、格式杂乱、没有稳定标题或段落标记；
- 需要**稳定块长**、方便控制 embedding 成本与上下文窗口；
- 流水线要求简单、可复现、延迟低。

**不太适合：**

- 强依赖章节结构的法律/技术文档（更建议层次或句界后再合并）；
- 单句极长且句末标点稀疏时，仍可能硬切。

### 2. 句子边界（`2-句子边界切片.py`）

**适合：**

- 中文/多语种**叙述性**文本、客服记录、转写稿；
- 希望块内以**完整句子**为主，减少「半句话」检索；
- 无 Markdown 标题、但标点相对规范。

**不太适合：**

- 表格、代码、列表项（句界不明显）；
- 英文缩写、小数点密集（可能被误当句末）。

### 3. LLM 语义切片（`3-LLM语义切片.py`）

**适合：**

- 结构复杂、规则难写清（多主题混排、口语与条目混杂）；
- 愿意承担 **API 成本与延迟**，希望切块「像人工分段」。

**不太适合：**

- 大批量离线建库且成本敏感（可用 1/2/4 替代或做抽样 LLM 校准）；
- 无可用 API Key 时（脚本会回退到句子边界模块）。

### 4. 层次切片（`4-层次切片.py`）

**适合：**

- **Markdown / 大纲式** 文档：`#` `##` `###` 清晰；
- 产品说明、Wiki、技术手册，需要**小节与正文不拆散**；
- 检索时希望块内带**小节标题**作上下文。

**不太适合：**

- 几乎没有标题的纯连续正文（退化为大块再句切，效果接近「先整篇再切」）；
- PDF 乱码转写、无 `#` 标记的扫描件。

### 5. 滑动窗口（`5-滑动窗口切片.py`）

**适合：**

- 担心**关键词跨两块边界**导致召回丢失，需要**重叠**；
- 对「语义块」要求不高，更关心覆盖率与实现简单。

**不太适合：**

- 需要严格不重复 token 的训练数据（重叠会带来冗余）；
- `step_size > window_size` 会漏字（脚本已禁止该配置）。

### 6. 自适应段落（`6-自适应切片.py`）

**适合：**

- 正文以**空行分段**（`\n\n`）较可靠：如新闻、博客导出；
- 段落长短不一，希望块长在目标值附近**自然浮动**。

**不太适合：**

- 单段超长、无内部分段（仍会并成超大块或触发硬切逻辑）；
- 只有单换行分段、没有空行（需先规范化排版）。

### 组合建议（简要）

- **RAG 通用默认**：固定长度 + 重叠，或句子边界 + 目标长度。  
- **文档型知识库**：层次优先，小节内超长再句界或固定长二次切。  
- **高难无结构**：抽样用 LLM 定边界规则，线上仍用规则切片降本。
